# 数据治理与来源策略

## 运行方式

系统只在用户发起分析时运行。它会先读取已验证缓存，再按字段的新鲜度补抓缺失内容；原始网页、来源 URL、抓取时间、内容哈希、适用日期和失败原因均会保存，避免重复抓取并支持复核。

这不是无人值守的实时行情任务。缓存用于可追溯与减少重复网络请求，不代表系统会持续后台采集。

## 固定取数顺序

1. 搜索或已知 URL 仅用于发现原始页面；
2. A 级官方页面优先确认事实；
3. B 级行情平台补充交易字段；
4. C 级权威媒体只作为事件线索，未经确认不进入评分；
5. 对目标 ETF、前十大权重、异常成交或评分临界数据执行交叉核验；
6. 本地确定性代码计算指标与评分，LLM 只解释已验证结果。

### 抓取执行边界

- Firecrawl 是公开网页行情、公告与新闻页面的默认后台抓取器，不需要启动本机 Chrome；
- Chrome 只用于后台抓取失败后的页面诊断、需要现有登录状态的内容，或用户明确提出使用 Chrome 时；
- 系统不会为了抓取数据自动启动 Chrome，也不会将 Chrome 页面可见内容绕过来源、日期和字段校验直接写入评分数据。

## 字段级规则

| 字段 | 首选 | 备用/校验 | 失败处理 |
|---|---|---|---|
| 基金身份、正式净值 | 基金公司官网 | 天天基金、东方财富比对 | 标记缓存日期或缺失 |
| 指数成分、权重 | 中证指数 | 无平台替代 | 不使用不完整名单做全量广度 |
| 当日 OHLCV、成交额 | 东方财富原页（MCP） | 同花顺原页；交易所收盘核验 | 重抓、备用页、记录失败；不以历史值替代 |
| ETF 份额、申赎、公告 | 交易所 | 东方财富补充 | 缺失即不计算变化 |
| 公司公告与政策 | 巨潮、交易所、政府、公司 IR | 权威媒体线索 | 未确认不计分 |
| 产业周期 | 官方统计 | WSTS、SIA 等产业机构 | 标注发布日期或缺失 |

## 冲突与缺失

- 同一交易日的价格、成交额不跨来源拼接；
- 官方来源与其他来源冲突时，保留冲突记录并优先官方；
- 份额变化、净申赎、主力资金和成交额是不同概念，禁止相互替代；
- 缺失字段不得以 0、经验值或“收盘价 × 成交量”补齐；
- 当日收盘数据需要完整 OHLCV/成交额、来源日期匹配和收盘状态确认；否则仅能作为盘中观测或不可用数据。
