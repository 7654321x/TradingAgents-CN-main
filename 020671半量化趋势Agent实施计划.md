# 020671 半量化趋势 Agent 实施计划

## 当前原则

- 基于现有 TradingAgents 项目扩展，不建立平行系统；
- 复用现有数据库、行情数据、基金报告、LLM 客户端和 CLI；
- 先保证数据日期、来源和统计口径正确，再实现评分和 LLM 解释；
- 不连接券商、不自动下单、不执行真实交易；
- 当前项目版本为 `1.0`，本任务完成后按发布规则进入 `1.1`。

## 第三阶段：确定性量化指标

### 目标

解决“ETF放量不等于板块放量”的统计口径问题。

### 实现内容

- 目标 ETF OHLCV；
- 指数成分股 OHLCV；
- 均线、RSI、MACD、ATR、收益率和回撤；
- ETF成交额相对5日、20日均值；
- 完整指数成分股合计成交额；
- 成分股上涨/下跌家数；
- 上涨成交额占比；
- 下跌成交额占比；
- 权重加权收益；
- 等权平均收益；
- 产业链分类和细分方向表现；
- 前十大权重贡献。

### 验收标准

- 报告同时展示 ETF 成交额和板块成分成交额；
- 不再将 ETF 单独放量写成板块放量；
- 10只基金披露股票不再替代完整指数成分。

依赖：第二阶段。

### 验收标准

- 趋势评分与短线评分分离；
- 缺失数据不会被填0；
- 报告能说明得出结论的具体依据。

依赖：第三阶段。

## 第七阶段：DeepSeek LLM 解释层

### 目标

在不修改现有 LLM 客户端代码的前提下，让项目自身的 DeepSeek 分析系统解释量化结果。

本阶段只新增基金分析侧的解释适配层，不修改以下目录中的实现：

- `tradingagents/llm_clients/openai_client.py`；
- `tradingagents/llm_clients/factory.py`；
- `tradingagents/llm_clients/capabilities.py`；
- `tradingagents/llm_clients/validators.py`。

接入必须通过现有 `create_llm_client(provider, model)` 工厂完成，沿用 `.env` 中已经配置的 Provider、模型和 API Key。

### 实现内容

- 新增基金分析侧的 LLM 解释模块，例如 `tradingagents/extensions/sector_fund/fund_llm_explainer.py`；
- 将量化 JSON、数据来源、板块成交额、广度、评分、历史样本和事件摘要传给 DeepSeek；
- 生成趋势解释、短线解释、多空因素、风险说明、数据限制和工具适配说明；
- 在同一份 Markdown 报告中追加“LLM 综合解读”分区；
- 在同一份 JSON 报告中追加 `llm_analysis` 对象；
- 增加显式 `--llm-explain` 参数，避免默认产生 API 费用；
- LLM 不可用时保存完整量化报告，并将 LLM 状态标记为 `unavailable` 或 `failed`。

建议 `llm_analysis` 至少包含：

```text
status, provider, model, generated_at, summary
trend_interpretation, short_term_interpretation
bullish_factors, bearish_factors, risk_warnings
data_limitations, tool_suitability, disclaimer, error
```

### 硬性约束

- LLM不能修改分数；
- LLM不能修改概率；
- LLM不能填补缺失字段；
- LLM不能把传闻写成事实；
- LLM不能生成确定买卖指令。
- LLM不能把 ETF 成交额写成板块成交额；
- LLM不能覆盖原始量化字段；
- LLM不能因为分析失败阻断量化报告生成。

### 验收标准

- 关闭 LLM 后量化报告仍可独立运行；
- LLM 输出能追溯到结构化输入；
- LLM 失败不会影响量化报告生成；
- Markdown 和 JSON 均能看到 LLM 状态；
- 量化分数、概率和数据源字段保持原值；
- API Key 不出现在报告、日志或错误信息中。

调用示例：

```bash
python -m tradingagents.storage.cli generate-fund-report \
  --fund-code 020671 \
  --analysis-date YYYY-MM-DD \
  --mode database_first \
  --llm-explain \
  --overwrite
```

依赖：第四、第五、第六阶段。
