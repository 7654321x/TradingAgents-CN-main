## 2. 已实现内容

### 2.1 多 Agent 决策流程

- 基于 LangGraph 编排市场、基本面、新闻、情绪等分析节点；
- 包含多头/空头研究、交易员决策和激进/中性/保守风险辩论；
- 支持状态传播、条件路由、信号处理、检查点恢复与反思记忆；
- Agent 输出提供结构化 Schema 和结构化调用辅助层。

### 2.2 数据访问与标的处理

- 实现 yfinance 日线、新闻、基本资料及财务数据访问；
- 实现 Alpha Vantage、FRED、Reddit、Stocktwits、Polymarket 等数据适配；
- 提供股票代码标准化、安全路径组件和日期边界处理；
- 提供行情数据质量校验、陈旧 OHLCV 防护和无数据错误处理；
- 包含 A 股 `.SS`、`.SZ` 代码路径和单 ticker/MultiIndex 兼容逻辑。

### 2.3 技术分析和证据体系

- 基于 Pandas/stockstats 计算并组织技术指标；
- 实现股票证据引擎、证据审计和规则化决策策略；
- 实现单股决策报告生成，包含趋势、风险、数据覆盖率和改变判断的条件；
- 已保存并纳入 Git 的示例报告：
  `reports/stocks/001309.SZ/2026-07-17/decision_report.md`。

### 2.4 基金扩展

- 提供基金持仓 Universe 种子数据与 2026Q1 持仓扩展；
- 实现基金技术报告和基金持仓存储相关逻辑；
- 基金生成结果默认位于 `reports/funds/`，按安全规则不批量提交生成文件。

### 2.5 数据库存储与迁移

- 使用 SQLAlchemy 2.x 风格的 Session、模型、仓储和数据服务；
- 提供市场数据、基金持仓等存储接口；
- 包含 Alembic 基线及当前存储 Schema 迁移；
- 本地 SQLite 数据库只用于运行和验证，已从 Git 排除。

### 2.6 LLM 适配

- 包含 OpenAI、OpenAI-compatible、DeepSeek、Anthropic、Google、Azure 和 Bedrock 客户端路径；
- 实现模型目录、Provider 路由、能力声明、参数校验和 API Key 环境变量解析；
- 覆盖 reasoning effort、temperature、重试、Responses API Base URL 等配置路径；
- 真实 API Key 仅保存在本地 `.env`，不会上传。

### 2.7 工程化

- 提供 CLI、Docker、GitHub Actions 和 pytest 配置；
- 当前包含 59 个 `test_*.py` 测试文件；
- 已编写项目架构文档 `架构.md`；
- 已增加 `.agent`，记录后续 Git 上传和安全规则。

## 4. 已知问题与限制

### 4.1 完整测试尚无最终结果

执行 `python -m pytest -q` 时，命令在约 124 秒达到外部超时，没有返回最终统计。不能据此声明完整测试全部通过。核心子集 89 项测试已通过，不受该超时结论影响。

后续建议：在不受 120 秒限制的终端运行完整测试，并保存最终通过、失败和跳过数量。

### 4.2 远端历史已切换到新的 1.0 基线

覆盖前的远端 `main` 是另一套包含 2173 个文件的工程。本次按仓库所有者明确要求，以当前本地项目作为 `1.0` 基线覆盖主分支。旧历史仍可通过覆盖前提交 `e695002ff727b86a7ca32293b13c2a07a6f578e6` 定位，但不再是默认主线。

### 4.3 生成报告默认不被跟踪

`.gitignore` 默认忽略 `reports/`，避免大量运行结果和潜在敏感数据进入仓库。本次示例报告通过 `git add -f` 单独加入。后续新增报告也必须逐份检查、逐份加入。

### 4.4 外部数据和 LLM 测试具有环境依赖

yfinance、新闻、FRED、社交媒体、DeepSeek 等真实请求依赖网络、Provider 状态、限流、代理和本地 `.env`。单元测试通过不等于所有外部服务在任意时间均可用。

### 4.5 决策报告不是投资建议

示例报告的数据质量为 `PARTIAL`，基本面和新闻证据覆盖不足，模型置信度未经历史校准。报告是研究信号，不是自动下单指令，也不构成投资建议。
